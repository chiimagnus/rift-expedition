# 20 · 技术架构（Web / Phaser / TypeScript）

> 归属：[E · 技术工程](README.md) ← [设计蓝图](../README.md)
> 本页方法论：technical-architecture（分层/单向数据流）+ data-driven-design（引擎有限、内容无限）+ game-perf（Web 性能）。
## 一、技术栈
- **语言**：TypeScript（严格模式，`tsc --noEmit` 做类型门禁）。
- **渲染/引擎**：Phaser 3（pixelArt 配置），通过 **CDN 加载**（沙箱未装 Phaser/PixiJS）。
- **构建**：esbuild / webpack 打包为静态站点，交付"一个链接"。
- **测试**：`node --test`（服务与模型层）＋平衡模拟 CLI。
- **运行时**：node 24.x 本地开发；目标浏览器见 [F/25 待决 Q4]。
## 二、分层架构（依赖单向：ui → viewmodels → services → models）
```plain text
entrypoints/main.ts   // 组装场景、注入依赖，可依赖所有层
  └─ ui/             // Phaser Scene、HUD：只读渲染 + 上抛 Command
      └─ viewmodels/ // 表现状态、把 Command 翻译成 service 调用
          └─ services/ // 战斗结算、寻路、AI、存档（纯逻辑，可测）
              └─ models/ // BattleState、Unit、Skill、Terrain（纯数据）
```
- **路径别名**（tsconfig + 打包器 + 测试三处同步）：`@ui/* @viewmodels/* @services/* @models/*`。
- UI 只读渲染 `BattleState`，输入 → 回调 → ViewModel → Services → Models，单向数据流。
### 分层职责边界（严守）
| 层 | 能做 | 禁止 |
| --- | --- | --- |
| ui | 渲染只读状态、播放演出、收集输入 | 写规则、直接改 models |
| viewmodels | 组织表现状态、把 Command 派发给 services | 含 Phaser 依赖、写战斗规则 |
| services | 结算/寻路/AI/存档等纯逻辑 | 引用 Phaser、读 DOM |
| models | 纯数据结构与不变量 | 含行为副作用 |

判定收益：services/models 不依赖 Phaser，可在 node 直接测（见 [E/22]）。
## 三、Phaser 配置（与 [C/15] 对齐）
```plain text
{ type: AUTO, pixelArt: true, roundPixels: true, antialias: false,
  scale: { mode: FIT, zoom: 整数 }, scene: [BootScene, BattleScene, UIScene] }
```
- BattleScene 只负责渲染只读状态与播放演出；不含战斗规则。
### 场景划分
- **BootScene**：加载图集/音频清单，显示进度条。
- **BattleScene**：渲染战棋网格与单位、播放战斗演出。
- **UIScene**：叠加 HUD（独立 Scene，避免相机缩放影响 UI）。
- **CutsceneScene**：解释 [B/14] 演出 DSL。
## 四、核心算法（复用 + 新增）
| 模块 | 方案 | 复用来源 |
| --- | --- | --- |
| 寻路/可达域 | 自研 Dijkstra/A\*（按移动消耗） | 复用裂隙远征 `reachable`/`pathTo` |
| 敌方 AI | Minimax + α-β 剪枝（可调深度） | 新增 TS 实现 |
| 回合流程 | 轻量 TS 状态机（部署→我方→敌方→结算） | 新增 |
| 随机数 | 可注入种子 PRNG（mulberry32/xorshift128） | 新增，保证可复现 |

### AI 分级（按敌种，呼应 [A/07]）
- **杂兵**：贪心（打最脆/最近），浅层。
- **精英**：Minimax 深度 2\~3，考虑相克与地形。
- **Boss**：脚本化行为树 + 阶段触发（血线/回合），可含"必杀预警"。
## 五、输入（全平台）
- Phaser pointer（触屏/触控板/鼠标）＋ keyboard ＋ **Gamepad API**（手柄）统一映射为 **Command**。
- 一套 Command 抽象适配键鼠/触控/手柄，满足"全平台通用"（见 [C/17] 焦点导航）。
```typescript
type Command =
  | { t: 'selectUnit'; unitId: string }
  | { t: 'moveTo'; x: number; y: number }
  | { t: 'attack'; targetId: string }
  | { t: 'useSkill'; skillId: string; target?: Cell }
  | { t: 'endTurn' }
```
所有输入源都产出 Command → ViewModel 统一处理，新增输入设备不动核心逻辑。
## 六、Web 性能（game-perf）
- 图集分组懒加载（units/tiles/portraits/fx/ui，见 [C/15]）。
- 高频音效对象池（见 [D/19]）；粒子发射器复用。
- 大地图用摄像机剔除；避免每帧分配（对象复用、避免 GC 抖动）。
- 首屏只加载 Boot + 首章资源，其余按章预载。
> ⚙️ **平台已定为 Web + Phaser + TS**（取代早期 Apple/SpriteKit 方案）。遵循 TS 开发规范：分层单向依赖、别名三处同步、配置最小复用、写编译/测试/构建门禁与冒烟检查（见 [E/22]）。
