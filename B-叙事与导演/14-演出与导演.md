# 14 · 演出与导演（分镜 / 镜头 / 节奏 / 文本）

> 归属：[B · 叙事与导演](README.md) ← [设计蓝图](../README.md)
> 本页方法论：narrative-design（叙事节奏）+ game-feel（镜头与冲击感）+ data-driven-design（演出脚本数据化）。
## 一、过场类型（按成本/权重分层）
| 类型 | 呈现 | 用于 |
| --- | --- | --- |
| 对话过场 | 像素立绘 + 对话框 + 背景 | 日常剧情、支援会话（占比最高，最省） |
| 战棋内演出 | 单位地图走位/特写 + 文字 | 战中转折、增援登场 |
| 全屏演出 CG | 高分辨率像素大图 + 运镜 | 幕高潮、重大反转、结局 |

## 二、镜头语言（Phaser 相机可实现）
- **震屏 shake**：暴击、必杀、龙痕觉醒、Boss 登场。
- **推拉 zoom**：聚焦关键单位/表情；慢推营造压迫。
- **慢镜 time-scale**：致命一击、角色死亡瞬间。
- **色调 tint/flash**：龙化（红）、圣光（白）、冰雷（蓝）。
- **运镜平移 pan**：展示战场全貌/增援方位（配合"增援可见"原则）。
## 三、战斗演出节拍
```plain text
预备(蓄力/踏步) → 命中帧(冲击+震屏+粒子) → 结算(飘字/血条) → 收势(回位)
必杀/暴击 → 插入特写与慢镜；龙痕觉醒 → 全屏色调+专属动机(见 [D/18])
```
## 四、叙事节奏（narrative-design）
- 故事节点**对齐战斗强度曲线**（见 [A/08]）。
- 过场是**赢来的**：放在硬仗之后，而非之前。
- 重大揭示后给**一章机制喘息**去消化。
## 五、过场指令 DSL（数据化演出）
```typescript
type CutsceneCmd =
  | { cmd: 'bg'; image: string }
  | { cmd: 'enter'; who: string; pos: 'L'|'C'|'R'; sprite?: string }
  | { cmd: 'line'; who: string; text: string; emo?: string }
  | { cmd: 'shake'; intensity: number; ms: number }
  | { cmd: 'zoom'; target: string; scale: number; ms: number }
  | { cmd: 'tint'; color: string; ms: number }
  | { cmd: 'sfx'; id: string }
  | { cmd: 'bgm'; id: string; fade?: number }
  | { cmd: 'wait'; ms: number }
  | { cmd: 'choice'; options: { text: string; flag: string }[] }
type Cutscene = { id: string; steps: CutsceneCmd[] }
```
演出由统一的 CutscenePlayer 解释，随剧情数据一起序列化（见 [E/21]）——新过场＝新数据，不写一次性代码。
### 示例分镜（第15章·织命低语）
```plain text
bg 神殿内部 → bgm 织命主题(fade) → enter 宗座 C
→ line 宗座 "你们以为在拯救世界？" → tint 红 300ms + shake
→ line 宗座 "从头到尾，都是我的棋局。" → zoom 宗座 1.4
→ wait 500 → line 奥德里克 "……神？"
```

## 七、文本规范
- 对话每句 ≤ 40 字，适配对话框；旁白克制。
- 命名遵循 [B/10] 阵营风格；术语统一（龙痕/龙化/织命神）。
- 情感高潮少用形容词堆砌，靠动作与留白（"演出而非陈述"）。
- 中文全角标点；语气区分：兄（沉稳）/妹（炙烈）/教会（庄严冷酷）。
- **本地化预留**：文本走 key → 文案表，不硬编码（为多语言预留，见 [E/21]）。
