# Rift Expedition — 内容型升级记录（2026-07-12）

本轮继续遵循“单人开发 + AI 协作 + 非成熟引擎”的现实边界，避免盲目堆叠 3A 式 UI 与超量系统，优先投入到更能直接提升可玩感与完成度的部分：

## 本轮重点

1. **原创角色立绘接入**
   - 新增 4 张可玩职业立绘：战士 / 弓手 / 法师 / 潜行者。
   - 立绘在 `RiftActorArt` 中接入，优先用于队伍创建、编队与其他角色展示位。
   - 保留原精灵表作为回退资源。

2. **环境关键画接入**
   - 新增：村庄广场、河湾废墟、矿洞深处 3 张章节关键画。
   - 主菜单远征简报、章节完成页已经接入关键画，以增强章节识别度与世界观氛围。

3. **分层音效与动态音乐**
   - `AudioService` 升级为多总线结构：主音量 / 音乐 / 环境 / 战斗与反馈音效。
   - 新增探索与战斗音景切换接口，支持基础音乐层 + 附加层 + 环境层。
   - 增加轻量合成音频资源，适合作为 solo 项目可替换的生产级占位资产。

4. **更多战斗动画反馈**
   - 为战斗表现事件新增 `BattleEffectStyle`：`strike / projectile / arcane / fire / poison / heal`。
   - 场景层会根据技能风格生成不同的粒子爆发与颜色反馈，让同样的回合制动作更易读。

5. **更多章节内容**
   - 新增 2 条第一章支线：
     - `灰烬中的誓言`（梁铮）
     - `矿工的最后一班`（村口守卫）
   - 新增 1 场遭遇：`河湾毒潮`
   - 修正任务交付逻辑：需要实际持有任务物品后，相关 NPC 才会出现交付对话。

## 本轮的有意克制

- 没有继续堆更多纯装饰型 UI。
- 没有引入难以维护的大型过场系统。
- 没有引入对当前工程体量不友好的“准 3A”内容负担。

## 下一轮建议（仍保持单人可承受）

1. 给 2～3 个关键 NPC 增补立绘（村长 / 梁铮 / 贺锋）。
2. 为 Boss 战加入一到两个专属 telegraph 动画。
3. 把章节 2 做成“1 个中心聚落 + 2 个野外区 + 1 个高潮区域”的同等规模切片。
4. 若音频资源继续扩展，优先给每个区域补一条更明显的高张力叠层，而不是增加更多 UI 点击音。
