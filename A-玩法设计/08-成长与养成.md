# 08 · 成长与养成（progression-systems）

> 归属：[A · 玩法设计](README.md) ← [设计蓝图](../README.md)
> 本页方法论：progression-systems（曲线类型 / 心流通道 / 难度三明治 / 解锁节奏）+ motivation-design（体感力量 vs 实际力量）。
## 一、成长的三条曲线
| 曲线 | 形状 | 体验 | 用在哪 |
| --- | --- | --- | --- |
| 线性 | 匀速上升 | 稳定、可预期 | 基础属性成长 |
| 递减 | 先快后缓 | 早期爽、后期需策略 | 等级经验需求 |
| 阶跃（S） | 平台+跃升 | 转职/觉醒的高光 | 转职、龙痕觉醒 |

混合使用：属性走线性、经验需求走递减、职业力量走阶跃 → 长期动力与高光并存。
## 二、体感力量 vs 实际力量
- **实际力量**：数值上真正的战力增长。
- **体感力量**：玩家“觉得自己变强”的感受——来自解锁新战术、秒掉旧敌、画面/音效反馈。
> 设计准则：两者都要给。纯数值涨（实际）但打法不变（体感缺失）会让成长“无味”；反之空有炫技无实力则挫败。转职给两者叠满。
## 三、经验与升级公式（候选）
```plain text
方案A 递减经验：nextExp(L) = base * L^1.5
击杀经验：killExp = base + max(1, 敌等 − 我等) * k   // 打高级敌给更多，抑制刷低级
辅助经验：治疗/增益/破防也给经验，避免“抢人头”
boss经验：一次性大额 + 阶跃解锁
```
所有系数进 GROWTH 可调参表，供 [E/21] 序列化、[E/22] 校验。
## 四、心流通道清单（progression-systems）
- [ ] 难度与玩家能力同步上升，保持在“挑战≈技巧”的通道内
- [ ] 每 2–3 章给一次能力跃升（转职/新技/新单位）打破平台期
- [ ] 失败可读、可快速再来（低惩罚重试），维持心流不断裂
- [ ] 提供“简单胜利”回落点（支线/低难度图）给挫败玩家喘息
## 五、难度三明治（三层难度并存）
| 层 | 面向 | 手段 |
| --- | --- | --- |
| 底层保底 | 休闲/剧情党 | 可选简单模式、成长保底、支援夹击 |
| 中层主线 | 核心玩家 | 行为难度、相克运用、地形博弈 |
| 顶层挑战 | 硬核/复玩 | 永久死亡、限турн、隐藏Boss、复玩加码 |

## 六、解锁节奏清单
- [ ] 新兵种/职业随章节梯度投放，不一次倾泻
- [ ] 每个新系统先在低压场景教学，再进硬仗
- [ ] 羁绊/支援会话随进度解锁，牵引养成×叙事
- [ ] 商店/锻造/道具库随资源规模同步开放
## 七、常见陷阱（规避）
> ⚠️ **规避成长反模式**：① 纯数值膨胀（打法不变）；② 一次性倾泻全部系统（认知过载）；③ 惩罚性经验（打高级敌反而亏）；④ 平台期过长无跃升（枯燥）；⑤ 后期数值破表使相克失效（呼应 [A/09]）。
