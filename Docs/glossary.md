# 术语表

给不熟悉游戏开发 / Swift 技术说法的人看的大白话说明；精确定义以代码为准。

| 术语 | 大白话解释 |
| --- | --- |
| AP（行动点） | 回合制战斗里角色每回合能花的“行动次数”，移动、放技能、用道具都要花 AP。 |
| Actor | 游戏里“能上场的角色”的统一叫法；玩家、NPC、敌人在代码里都是同一种 Actor，只是阵营、职业、等级不同。 |
| Faction（阵营） | 区分敌我的标签，比如“玩家方”“敌方”“中立”。 |
| Effect（技能效果） | 技能具体做的事情，比如造成伤害、治疗、加状态、生成地表、位移等；一个技能可以组合多个 effect。 |
| 地表（Surface） | 地图上一块会持续影响角色的区域，比如火焰、毒雾、油渍，通常由技能触发生成。 |
| navObstacle | Tiled 地图里标记“这里不能走 / 挡视线”的区域，比如墙、箱子。 |
| spawn / npc / encounter / trigger / exit / item | Tiled 地图里几种固定用途的标记层：出生点 / NPC 位置 / 战斗触发范围 / 事件触发器 / 出口传送点 / 掉落物或拾取物。 |
| schemaVersion | 存档文件里记录的“格式版本号”，方便以后升级存档格式时知道怎么兼容旧存档。 |
| self-made / CC0 / ai-static（资源授权类型） | self-made＝自己做的；CC0＝任何人都能免费商用、不用署名的公共资源；ai-static＝AI 生成的静态图片（不是成套动画）。 |
| ponytail 注释 | 代码里故意留的“技术债”标记，写明“这里是图省事的写法，以后可能要重构”，方便以后集中处理，不代表是 bug。 |
| classID / kind | 分别是“人类敌人的职业类型”（比如弓手 / 近战 / 精英）和“野兽敌人的种类”（比如普通动物 / 受污染生物）的内部标记，用来决定用哪张立绘。 |
